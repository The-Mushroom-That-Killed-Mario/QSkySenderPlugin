Инструкция по использованию лагином QSkySenderPlugin.jar для сбора статистики

Для начала требуется получить доступ к серверу статистики. Пароль и имя пользователя выдается администратором(info@apertum.ru). Далее по этому паролю и пользователю заходим на сервер. У вас уже есть один филиал и у него есть идентификатор. Наименование и идентификатор можно менять. Идентификатор потребуется указать в аминке далее в п.3.2. Филалы можно добавлять и удалять выстраивая древовидную структуру.


1. Плагин QSkySenderPlugin.jar помещаем в папку plugins для сервера очереди и для админки, если админка находится отдельно от сервера.
2. Открываем админку на вкладке "Сеть и таймауты".
3. Настраиваем "Плагин облачного сервиса".
3.1. URL указываем http://dev.apertum.ru:8080/qskyapi/customer_events?wsdl
3.2. Номер филиала в облаке укажите такой, какой сами выбрали для своего филиала. Выберете как войдете на страницу облачного сервиса, либо оставте какой есть.
4. Сохраняем изменения в админке. Рестарт админки на всякий случай.
5. Проверяем доступность сервера. Жмем "Тест облака". Ответ должен быть "1".
6. Далее синхронизируем услуги и пользователей с сервером. Жмем "Обновить данные облака". Названия услуг и пользователей должны синхронизироваться, что должно быть видно наглядно в виде счетчика. Можно не выполнять эту синхронизацию. В этом случае услуги и пользователи будут отражены абстрактным идентификатором.
7. Вот и все. Теперь все изменения очереди онлайн будут отправляться в центр статистики после рестарта сервера очереди.
